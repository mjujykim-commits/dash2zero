### 2.2 자율 결정: Detox vs Maestro hybrid

**최종 권고 (qa 자율, devops는 인프라 의견만)**:
- **Maestro 메인 (P0 7건 + P1 4건)** — 단순 tap-tap-assert 흐름 + flaky 적음 + YAML이 PM/디자이너 검토 가능
- **Detox 결제+a11y suite (P0 5건 + P1 2건)** — StoreKit Configuration + RC sandbox webhook의 retry/timeout 표현력 + iOS XCUITest accessibility query 정밀도

**근거** (가중치 매트릭스 11축):
1. Maestro는 RN/Expo flaky 적음 — baseline metric 회귀 오인 위험 회피 (readiness §4 risk #9 인접)
2. 결제 시나리오는 JS `await`+retry가 YAML보다 표현력 높음
3. a11y는 두 도구 모두 한계 — Detox + iOS XCUITest accessibilityLabel query가 Maestro보다 정밀
