## 2. 핵심 산출물 상세

### 2.1 `merge-guest` Edge Function (CC2-04 + BE-NEW-003)

- 인증 사용자 검증 → device_install_id 기반 idempotency 체크
- 이미 다른 user_id로 머지된 디바이스: 409 Conflict
- attempts UPSERT (UNIQUE on (user_id, client_attempt_id) — append-only)
- user_word_states 충돌 해결: server vs guest의 last_attempt_at 비교 → 더 최근 채택 + counts 합산
- profile.merged_at + guest_sessions.merged_to_user_id + audit_log
- 알려진 한계: SRS stage 정확 재계산은 attempt 시퀀스 replay 필요 (M3 보강), 현재는 단순 count 추정

### 2.2 `content-manifest` Edge Function (CC2-15 + BE-NEW-005 + FE-NEW-010)

- 익명/인증 분기:
  - anon: starter pack only (CC2-09 결정)
  - free authenticated: starter + Premium pack의 free_sample 10 (CC3-01 preview pool)
  - premium: 모든 pack
- ETag 헤더 + If-None-Match 304 (FE-DOC-010)
- `since` 쿼리로 diff fetch
- Cache-Control: 5분 private cache

### 2.3 `audio-signed-url` Edge Function (CC3-04 + BE-NEW-006)

- free tier: public URL 영구
- premium tier: TTL 6시간 signed URL (CC3-04 결정)
- 미인증 + premium 요청 → 401
- 인증 + premium 미가입 → 403 `premium_required`
- retired audio → 410 `retired`

### 2.4 Supabase auth client + AuthClient 어댑터 (ADR-0002)

- expo-secure-store 어댑터 — RN 안전 저장
- `AuthClient` 인터페이스 5 메서드 구현 (Apple, Google, Magic Link send/verify, signOut, getCurrentUser, refreshSession)
- Phase 4 Auth0/Cognito 전환 시 본 파일만 교체

### 2.5 sign-in 화면 (CC-03 + CC3-03)

- Apple Sign In (M2-S6에 expo-apple-authentication 통합)
- Google Sign In (M2-S6 expo-auth-session)
- Email magic link (deep link `dash2zero://auth/callback` — FE-DOC-006)
- 비밀번호 미제공
- "Check your email" 안내 후 link 만료 60분

### 2.6 BOOTSTRAP_INFRA.md runbook

8개 영역 외부 작업 절차 봉인:
1. Supabase 3 프로젝트 (us-east-1, Free → Pro)
2. Google Cloud TTS (ko-KR-Neural2-A, ~$0.06)
3. RevenueCat (Webhook URL → revenuecat-webhook Edge Function)
4. Apple Developer / Google Play (Bundle/Package, IAP, Privacy Label, Territory)
5. Firebase Spark (BigQuery disabled — CC2-23)
6. 1Password Emergency Kit (CC2-21 dead man's switch)
7. 검수자 계약 (R-01 완화 — Reddit/Discord 모집)

### 2.7 API client + useTodaySummary
