## 2. 핵심 산출물 상세

### 2.1 SRS Leitner (`apps/mobile/src/srs/leitner.ts` + spec)

CC-08 / CC2-10 / CC3-05 결정을 코드로 정확히 반영:
- `STAGE_INTERVAL_DAYS = {1:1, 2:3, 3:7, 4:14, 5:30}`
- `applySrsTransition(state, attempt)` — pure function, 부작용 없음
- `localDay04(date, timezone)` — CC-17 04:00 로컬 자정 계산 (Intl API)
- 단위 테스트 12 case (golden 50의 핵심 분포: 정답 3 + 오답 3 + same-cycle 3 + due 2 + initial 1)

### 2.2 Edge Function `submit-attempt`

POST → user JWT 검증 → service_role로 RLS bypass → `learning_attempts` insert (idempotent) → `applySrs` → `user_word_states` upsert → `daily_usage` atomic increment (RPC) → 응답.

핵심 검증:
- 일일 한도 (CC2-07: 무료 3 / Premium 15 / review 20 / unlimited)
- entitlement 조회 (CC2-08 status enum)
- idempotency (UNIQUE on `client_attempt_id`)

### 2.3 Privacy Manifest (app.json)

CC2-04 + Q-FE-NEW-001 처리:
- `NSPrivacyTracking: false`
- `NSPrivacyAccessedAPIType` 2종 선언 (UserDefaults CA92.1, FileTimestamp C617.1)
- ATT 사용 안 함 (NSUserTrackingUsageDescription만 정직 명시)
