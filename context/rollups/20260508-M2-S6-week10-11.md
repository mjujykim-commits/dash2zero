## 2. 핵심 산출물 상세

### 2.1 revenuecat-webhook (CC2-08 + CC3-05)

- 시그니처 검증: `Authorization: Bearer <REVENUECAT_WEBHOOK_SECRET>` 일치
- RC 12 이벤트 → 9 status 매핑 (mapStatus 함수)
- TEST / SUBSCRIBER_ALIAS 무시 (200 OK)
- last_rc_event_id로 멱등 (재전송 무시)
- user_id 매칭: app_user_id / original_app_user_id / aliases UUID 형식 시도
- 매칭 실패 시 audit_log + 200 OK (RC retry 방지)
- subscription_entitlements UPSERT (user_id + product_id 기준 single row)
- 모든 변경 audit_log (action='webhook_revenuecat')

### 2.2 delete-account (CC-11 + CC2-11 + Q-BE-DOC-010)

- 30일 SLA: scheduled_hard_delete_at = now + 30일
- profiles.soft_deleted_at = NOW (즉시 차단)
- RevenueCat DELETE /v1/subscribers/{user_id} (best-effort)
- export_data 옵션 (M2-S7에서 비동기 worker)
- 사용자 세션 즉시 무효화 (admin.auth.signOut)
- 30일 내 재로그인하면 cancel 가능 (M5 UI 추가)

### 2.3 purchases.ts (CC-09 + CC2-06)

- `initPurchases(userId)` — 인증 사용자만 RC 초기화 (CC2-06)
- `fetchOfferings()` — monthly + annual 두 plan
- `purchasePackage()` + `restorePurchases()` 모두 analytics 이벤트 발화 (CC2-22)
- 가족 공유 비활성은 RC dashboard 설정으로 처리 (CC-09)

### 2.4 Paywall 화면 (DESIGN_DIRECTION §8.3 + Q-PL-NEW-005)

Honest 톤 disclosures:
- "No free trial."
- "Family Sharing is not supported."
- "Subscription auto-renews until canceled. Manage or cancel in your App Store / Google Play settings at least 24 hours before renewal."
- "Refunds follow App Store / Google Play policies."

CC2-06: 게스트 → 구매 시 sign-in 강제.

### 2.5 Settings 화면

5 섹션: Account / Subscription / Privacy / About / Danger.

- Restore Purchases (CC-09 필수)
- Privacy choices 재진입
- Privacy Policy / Terms 외부 링크
- Delete account (확인 + 30일 안내 + 즉시 sign out)

### 2.6 Report 화면 (CC2-15)

5 카테고리 라디오 + 선택 description (max 500자). RLS 정책에 따라 anon은 device_id, 인증은 user_id 자동.

### 2.7 useLesson hook + 게스트 머지 트리거

- content-manifest fetch → word_id 검색 → distractors 3개 무작위
- audio-signed-url로 word + example URL 받음
- in-memory manifest cache (M2-S7에서 SQLite 보강)
- triggerGuestMergeIfNeeded — sign-in 직후 호출 패턴
