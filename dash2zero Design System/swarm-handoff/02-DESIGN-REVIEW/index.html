Senior Design Review · 2026-05-26

한글이 주인공인 무대.
지금은 무대가 너무 정적이다.

dash2zero는 D-022 K-pop Bold 결정 이후 시각적 정체성을 확고히 다졌다. 다크 캔버스, 네온 그라데이션, 64–88px 한글 hero — 톤은 분명히 살아 있다. 그러나 사용자가 실제로 손을 대는 순간, 즉 인터랙션의 표면에서는 아직 "정적인 mockup"의 흔적이 남아 있다. 오늘 리뷰의 결론은 한 줄이다 — 시각은 A, 모션은 C. 그리고 그 간극을 메우는 작업은 모두 0.15–0.4초 안에 끝낼 수 있다.

03 · 치명적 약점

지금 손대지 않으면 GA에 발목 잡힐 다섯 가지

1

학습 단계 전환이 즉시 교체된다 P0

apps/mobile/app/lesson/[wordId].tsx의 stage 전환은 setStage() 호출 하나로 끝난다. Notice → Hear → Meaning → Retrieve는 학습 루프의 본체이고 사용자는 하루에 12번 그 전환을 본다. 그런데 지금은 모든 단계가 같은 위치에 펑 하고 나타난다. fade도 없고, 슬라이드도 없고, hero glyph의 위치 변화조차 없다.

→ 처방: 한글 글자는 좌상단으로 축소하면서 RR + gloss가 fade-up으로 들어오는 단일 시그니처 모션. 240ms ease-out, stagger 60ms.

2

정답·오답 피드백에 "리액션"이 없다 P0

현재 코드는 정답이면 success 보더, 오답이면 danger 보더로 색만 변경한다. docs/brand/DESIGN_DIRECTION.md §9.1의 "정답/오답 피드백은 색만 (모션 없음)"이라는 Quiet 시절 규칙이 D-022로 톤이 바뀐 뒤에도 그대로 살아 있는 것.

→ 처방: 정답은 success glow가 부드럽게 펄스 + ✓ icon이 0.4→1로 spring scale. 오답은 360ms shake + danger border. 학습 흐름을 끊지 않는 한도 안에서 리액션을 부활시키는 게 D-022 정신에 더 맞다. 아래 랩 §02에서 직접 눌러보고 판단하길.

3

버튼에 ripple도, hover lift도, focus ring도 없다 P1

NeonButton.tsx는 press에서 scale(0.96)만 적용. 웹 prototype CSS에는 hover translateY조차 없다. 키보드 사용자는 focus indicator 자체가 없는 상황(접근성 부채).

→ 처방: Press = 물결(ripple) + scale(0.96), Hover = translateY(-2px) + glow 강화, Focus = 2px 외곽 neon ring. 모두 200ms 이하. 아래 §1.1 라이브 데모 참고.

4

로딩 상태가 ActivityIndicator 회색 스피너 하나 P1

home.tsx 로딩 중 모습은 <ActivityIndicator color={lightColors["neon.cyan"]} size="large" />. 시스템 기본 UIActivityIndicatorView가 cyan으로 칠해진 것에 불과. D-022 톤과 무관한 외형이고, 사용자에게 "이 앱이 무엇을 로딩 중인지" 단서를 주지 않음.

→ 처방 둘 중 택일: (a) gradient conic spinner + neon glow, (b) skeleton — 카드 형태를 그대로 둔 채 shimmer 1.4s ease-in-out. State pattern 문서가 이미 skeleton을 권장하고 있으니 (b)가 정답.

5

streak·mastered 같은 "성취 표시"가 정적이다 P2

Lesson Complete에서 +3이 표시되는데 그냥 그 자리에 있다. 하루에 한 번 보는 화면이라면 숫자가 카운트업되며 등장하는 0.6초의 micro-moment를 만들 가치가 충분하다. 추가로 streak 뱃지에는 flame flicker 1.6s 알트네이트 — 보기엔 사소하지만 화면이 살아 있다는 신호.

→ 처방: StatCard.value를 data-counter 패턴으로 만들고 IntersectionObserver로 viewport 진입 시 1회 carryover. 아래 §4.2.

05 · 모션 토큰 처방

packages/design-tokens/src/motion.ts 권장 갱신

drop-in replacement

아래 토큰은 현재 4개(80/150/200/300ms)의 duration을 7개로 확장 + spring easing을 추가한 권장안. 모든 값은 위 랩에서 검증됨.

80ms
motion.tap
버튼 press scale, choice tap. 너무 짧으면 응답이 아예 없어 보이고, 100ms 넘으면 느린 느낌.
150ms
motion.fast
toggle color cross-fade, hover lift. 사용자가 의식하지 않을 만큼 빠른 변화.
200ms
motion.base
기본값. tab swap, modal backdrop, icon morph. 95%의 transition은 여기에 둘 것.
240ms
motion.stage
신규 · lesson stage 전환에 헌정. fade-out 240 → fade-in 240 stagger.
320ms
motion.spring
신규 · toggle knob, badge pop, ✓ scale-in. easing.spring과 짝.
360ms
motion.sheet
bottom sheet rise, slide page push. backdrop fade(220ms)와 의도적 어긋남.
600ms
motion.progress
progress bar fill, lesson chain advance. 값 변화가 큰 transition.
900ms
motion.count
counter tick-up. Lesson Complete · Mastered total. hero motion이므로 화면당 1회 제한.
Easing additions
easing.out      cubic-bezier(0.16, 1, 0.3, 1)      // 95% — defaults
easing.in       cubic-bezier(0.7, 0, 0.84, 0)      // exits
easing.in-out   cubic-bezier(0.65, 0, 0.35, 1)     // toggles, color shifts
easing.spring   cubic-bezier(0.34, 1.56, 0.64, 1)  // ✨ NEW — overshoot for pop / badge / knob
easing.shake    cubic-bezier(0.36, 0.07, 0.19, 0.97) // ✨ NEW — incorrect answer
