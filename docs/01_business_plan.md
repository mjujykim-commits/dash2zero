## 0. 문서 사용 원칙

이 문서는 투자자 설득용 문서가 아니라 **제품 개발 의사결정용 내부 기준 문서**다.

따라서 시장 규모 수치, 외부 리서치 인용, 장황한 트렌드 설명보다 다음 항목을 우선한다.

- 어떤 사용자를 먼저 공략할 것인가
- 어떤 문제를 해결할 것인가
- MVP에서 무엇을 만들고 무엇을 만들지 않을 것인가
- 1인 개발자가 감당 가능한 사업 모델은 무엇인가
- 후속 PRD, 기능 명세서, 화면설계서, 개발 태스크가 따라야 할 기준은 무엇인가

이 문서에서 확정된 내용은 후속 문서 작성과 개발 구현의 기본 전제로 삼는다. 단, 법무, 개인정보, 앱스토어 심사, 결제 정책은 별도 정책 문서에서 더 엄밀하게 확정한다.

## 1. Executive Summary

dash2zero는 **영어권 외국인 초보 학습자를 위한 모바일 중심 한국어 단어 학습 서비스**다.

기존 한국어 학습 앱은 종합 언어 교육, 단순 단어 암기, 회화 훈련 중 하나에 치우치는 경우가 많다. dash2zero는 그 사이에서 **한국어 초보자가 하루 3분 안에 한국어 단어를 보고, 듣고, 기억에서 꺼내고, 짧은 문장 패턴으로 이해하게 만드는 초경량 학습 루프**에 집중한다.

MVP는 음성 녹음, 발화 평가, 실시간 AI 튜터, AI 개인화 학습을 포함하지 않는다. 대신 **고품질 단어 카드, 원어민 또는 고품질 음성, 능동 인출 퀴즈, 간격 반복 복습, 상황 기반 예문, 영어권 사용자에게 친절한 UX**에 집중한다.

사업 모델은 **무료 + 저가 구독형 프리미엄**이다. 1인 개발자 운영을 전제로 하므로 가격은 경쟁 앱보다 낮게 설정하고, 기능 범위는 유지보수 가능한 수준으로 제한한다.

초기 제품 목표는 거대한 한국어 교육 플랫폼이 아니라 다음 한 문장을 실현하는 것이다.

> 영어권 초보자가 매일 3분 동안 한국어 단어를 실제 상황 속에서 기억하고 다시 꺼내게 만든다.

## 2. 사업 정의

### 2.1 한 문장 정의

dash2zero는 영어권 초보 학습자가 한국어 단어를 단순 번역으로 외우는 데서 벗어나, **한글 형태, 발음, 의미, 예문, 복습을 하나의 짧은 모바일 루프로 학습하게 만드는 한국어 단어 학습 앱**이다.

### 2.2 서비스명 해석

`dash2zero`는 학습자가 한국어를 시작할 때 느끼는 장벽을 빠르게 낮춘다는 의미로 사용한다.

- dash: 짧고 빠른 학습 세션
- 2: 목표 상태로 이동하는 경로
- zero: 한국어 진입 장벽, 막막함, 학습 부담을 0에 가깝게 낮추는 방향

브랜드 문장 후보:

> Korean words, down to zero friction.

한국어 내부 설명:

> 한국어 단어 학습의 막막함을 0에 가깝게 낮추는 3분 학습 루프.

### 2.3 제품 카테고리

dash2zero는 다음 카테고리에 속한다.

- 모바일 언어 학습 앱
- 한국어 초보자용 단어 학습 앱
- 마이크로 러닝 앱
- 간격 반복 기반 복습 앱
- K-콘텐츠 및 여행 기반 실용 한국어 입문 앱

MVP 단계에서는 회화 앱, AI 튜터 앱, TOPIK 시험 대비 앱, 종합 한국어 강의 앱으로 포지셔닝하지 않는다.

## 3. 문제 정의

### 3.1 사용자가 겪는 핵심 문제

영어권 초보자가 한국어를 시작할 때 겪는 문제는 단순히 단어 수가 부족한 것이 아니다.

| 문제 | 설명 | 제품 관점의 의미 |
|---|---|---|
| 한글 형태가 낯설다 | 단어가 알파벳처럼 직관적으로 분리되지 않고 덩어리처럼 보인다 | 단어 카드에서 한글 구조와 발음을 함께 보여줘야 한다 |
| 발음이 예측되지 않는다 | 표기와 실제 소리가 다르게 느껴질 수 있다 | 텍스트 중심 학습만으로는 부족하다 |
| 단어가 문장으로 이어지지 않는다 | `coffee = 커피`를 알아도 `커피 주세요`로 바로 전환하지 못한다 | 단어와 최소 문장 패턴을 함께 제공해야 한다 |
| 복습 체계가 약하다 | 처음 본 단어를 다시 만나지 못하면 빠르게 잊는다 | 간격 반복 기반 재노출이 필요하다 |
| 학습 부담이 크다 | 긴 강의나 문법 설명은 초반 이탈을 만든다 | 하루 3분 안에 끝나는 작은 루프가 필요하다 |
| 학습 동기가 감정적이다 | K-콘텐츠, 여행, 친구, 음식, 문화 접근 욕구가 크다 | 시험지형 콘텐츠보다 상황 기반 콘텐츠가 적합하다 |

### 3.2 재정의된 문제

dash2zero가 해결할 문제는 다음과 같다.

> 영어권 초보자가 한국어 단어를 낯선 문자 덩어리로 외우는 것이 아니라, 소리와 상황 속에서 이해하고, 짧은 문장 패턴으로 다시 꺼내게 만드는 문제.

### 3.3 MVP에서 풀지 않는 문제

MVP는 다음 문제를 직접 해결하지 않는다.

- 사용자의 발음을 자동 평가하는 문제
- 사용자의 음성을 녹음하고 분석하는 문제
- AI가 실시간으로 개인 튜터처럼 대화하는 문제
- TOPIK 고득점을 위한 문법, 독해, 듣기 종합 훈련 문제
- 중급 이상 학습자의 긴 문장 생산 문제
- 교사와 학생을 연결하는 교육 플랫폼 문제

이 범위를 벗어나면 1인 개발자가 MVP를 완성하기 어렵고, 제품의 핵심 학습 루프가 흐려진다.

## 5. 시장 기회

### 5.1 정성적 시장 판단

dash2zero가 노리는 시장은 이미 존재하는 언어 학습 시장 안의 작은 진입점이다.

이 시장의 기회는 거대한 종합 앱을 이기는 데 있지 않다. 오히려 영어권 초보자가 한국어를 처음 접할 때 느끼는 구체적인 불편을 더 날카롭게 해결하는 데 있다.

초기 시장 기회는 다음 네 가지에서 나온다.

1. K-콘텐츠를 통해 한국어를 가볍게 시작하려는 사용자가 계속 유입된다.
2. 많은 사용자는 긴 강의보다 짧은 모바일 루틴을 선호한다.
3. 한국어는 문자, 발음, 조사, 문장 순서 때문에 영어권 사용자에게 별도 설계가 필요하다.
4. 기존 앱들은 종합 언어 앱이거나, 단어 암기 앱이거나, 회화 앱으로 나뉘어 있어 초보용 단어-문장 연결 경험이 빈다.

### 5.2 dash2zero의 시장 진입 방식

dash2zero는 다음 방식으로 진입한다.

- 시장 전체를 공략하지 않는다.
- 영어권 초보자만 먼저 본다.
- 한국어 단어 학습만 먼저 본다.
- 하루 3분 루틴만 먼저 완성한다.
- 고급 기능보다 반복 사용성과 콘텐츠 품질을 우선한다.
- 낮은 가격으로 진입 장벽을 줄인다.

## 6. 경쟁 포지셔닝

### 6.1 경쟁 서비스 유형

| 유형 | 대표 예시 | 강점 | 약점 | dash2zero의 대응 |
|---|---|---|---|---|
| 종합 언어 학습 앱 | Duolingo 등 | 습관 형성, 게임화, 쉬운 진입 | 한국어 특화 깊이가 제한적일 수 있음 | 한국어 구조에 더 집중한다 |
| 이미지 기반 단어 앱 | Drops 등 | 빠른 단어 노출, 시각적 기억 | 문장 전환이 약할 수 있음 | 단어를 최소 문장 패턴과 연결한다 |
| 아시아 언어 특화 앱 | LingoDeer 등 | 문법과 구조 설명 | 초반 사용감이 학습지처럼 느껴질 수 있음 | 더 짧고 가벼운 모바일 루틴으로 간다 |
| 회화 앱 | Teuida 등 | 실제 상황 말하기 경험 | 단어 축적과 복습이 부차적일 수 있음 | 단어 축적과 복습을 핵심으로 둔다 |
| 챗봇형 학습 앱 | Eggbun 등 | 대화형 친밀감 | 챗봇 품질과 운영 부담이 큼 | MVP에서는 챗봇을 하지 않는다 |
| 공공 교육 앱 | 세종학당 계열 | 공신력, 체계성 | 재미와 반복 사용성이 약할 수 있음 | 초보 친화적 사용감과 짧은 루틴으로 차별화한다 |

### 6.2 경쟁하지 않을 영역

dash2zero는 MVP에서 다음 영역의 1등을 목표로 하지 않는다.

- 가장 많은 단어 수
- 가장 많은 문법 강의
- 가장 정교한 발음 평가
- 가장 똑똑한 AI 튜터
- 가장 화려한 게임화
- 가장 완전한 한국어 커리큘럼

### 6.3 이길 수 있는 작은 영역

dash2zero가 초기부터 이겨야 할 영역은 다음 하나다.

> 영어권 초보자가 매일 3분 동안 한국어 단어를 부담 없이 다시 열어보게 만드는 경험.

이를 위해 제품은 다음 감각을 줘야 한다.

- 오늘 할 분량이 작다.
- 단어가 눈에 들어온다.
- 발음을 들을 수 있다.
- 뜻을 기억에서 꺼내보게 된다.
- 짧은 예문 안에서 단어가 쓰이는 방식이 보인다.
- 어제 배운 단어가 적절한 타이밍에 다시 나온다.

## 7. 제품 전략

### 7.1 핵심 제품 약속

MVP의 제품 약속은 다음과 같다.

> 하루 3분, 한국어 단어 3개를 보고·듣고·기억하고·문장 속에서 이해한다.

기존 논의의 "말하기" 방향성은 장기적으로 유지하되, MVP에서는 음성 녹음과 발화 훈련을 제외한다. 따라서 MVP의 초점은 **말하기 평가**가 아니라 **단어 이해, 인출, 문장 연결, 복습**이다.

### 7.2 핵심 학습 루프

MVP 학습 루프는 다음 순서로 설계한다.

1. Notice: 한글 단어와 구성 요소를 본다.
2. Hear: 단어 발음을 듣는다.
3. Meaning: 영어 의미와 상황 이미지를 통해 뜻을 이해한다.
4. Retrieve: 영어 뜻 또는 이미지를 보고 한국어 단어를 고른다.
5. Context: 짧은 예문 안에서 단어가 어떻게 쓰이는지 본다.
6. Review: 다음 세션에서 적절한 시점에 다시 만난다.

### 7.3 MVP 차별화 요소

| 차별화 요소 | 설명 | MVP 반영 방식 |
|---|---|---|
| 한국어 특화 카드 | 한글, 로마자, 발음, 의미, 예문을 함께 제공 | 단어 카드 기본 구조로 반영 |
| 초단기 루틴 | 하루 3분 안에 끝나는 학습 | 일일 학습 단어 수와 화면 흐름 제한 |
| 능동 인출 | 보기만 하지 않고 기억에서 꺼내게 함 | 객관식, 매칭, 빈칸 퀴즈 |
| 상황 기반 콘텐츠 | K-콘텐츠, 여행, 음식, 일상 상황에 연결 | 단어팩과 예문 설계에 반영 |
| 간격 반복 | 잊기 전에 다시 나오게 함 | 단어별 학습 상태와 복습 우선순위 |

### 8.3 MVP 완성 기준

MVP는 다음 조건을 만족하면 완성으로 본다.

- 영어권 초보자가 계정 생성 또는 게스트 진입 후 첫 학습을 3분 안에 완료할 수 있다.
- 최소 300개 단어 콘텐츠가 앱에서 정상적으로 제공된다.
- 각 단어는 한글, 로마자, 영어 뜻, 발음, 예문, 태그를 가진다.
- 학습 결과가 저장되고 다음 복습에 반영된다.
- 무료 사용자가 제품 가치를 체험할 수 있다.
- 프리미엄 사용자는 더 많은 콘텐츠와 복습 편의성을 얻는다.
- 앱스토어와 구글플레이에 제출 가능한 수준의 안정성을 갖춘다.

## 9. 콘텐츠 전략

### 9.1 초기 콘텐츠 범위

초기 콘텐츠는 **300단어**를 기준으로 한다.

| 구분 | 단어 수 | 목적 |
|---|---:|---|
| 생존 한국어 | 100 | 인사, 감사, 요청, 숫자, 장소, 이동 등 |
| 생활 한국어 | 100 | 음식, 카페, 쇼핑, 편의점, 날씨, 감정 등 |
| K-콘텐츠/관계 표현 | 100 | 드라마, 팬 활동, 친구, 감정, 일상 표현 등 |

### 9.2 단어 선정 기준

단어는 다음 기준으로 선정한다.

- 영어권 초보자가 실제로 자주 마주칠 가능성이 높은가
- 한국 여행이나 K-콘텐츠 이해에 바로 도움이 되는가
- 짧은 예문으로 사용 방식을 보여줄 수 있는가
- 비슷한 단어와 비교 학습 가치가 있는가
- 지나치게 시험용이거나 추상적인 단어는 아닌가
- 발음, 받침, 조사 연결 등 한국어 특성을 보여줄 수 있는가

### 9.3 단어 데이터 기본 구조

각 단어는 최소한 다음 정보를 가진다.

| 필드 | 예시 | 설명 |
|---|---|---|
| korean | 사과 | 한글 표기 |
| romanization | sagwa | 영어권 초보자용 로마자 |
| english | apple | 영어 의미 |
| part_of_speech | noun | 품사 |
| level | A0 | 난이도 |
| category | food | 카테고리 |
| tags | travel, cafe | 검색/추천용 태그 |
| example_ko | 사과 주세요. | 짧은 한국어 예문 |
| example_en | Apple, please. | 예문 번역 |
| audio_word | word audio path | 단어 음성 |
| audio_example | example audio path | 예문 음성 |
| distractors | 바나나, 물, 커피 | 퀴즈 오답 후보 |

### 9.4 AI 활용 원칙

MVP에서 AI는 사용자 기능이 아니라 **콘텐츠 제작 보조 도구**로만 사용한다.

AI가 도울 수 있는 작업:

- 예문 초안 생성
- 단어 난이도 후보 분류
- 카테고리 태깅 초안 생성
- 오답 후보 초안 생성
- 영어 설명 초안 생성

반드시 사람이 검수해야 하는 작업:

- 한국어 예문 자연성
- 영어 번역 정확성
- 문화적으로 부적절한 표현 여부
- 초보자 난이도 적합성
- 발음과 로마자 표기 적합성

### 10.3 무료 플랜 범위

무료 플랜은 다음을 제공한다.

- 기본 온보딩
- 매일 제한된 신규 단어 학습
- 기본 복습
- 일부 무료 단어팩
- 기본 성장 지표

무료 플랜은 너무 좁게 막지 않는다. 사용자가 학습 효과를 느껴야 유료 전환이 가능하다.

### 10.4 프리미엄 플랜 가치

프리미엄은 다음 가치를 제공한다.

- 전체 단어팩 접근
- 일일 학습 제한 완화 또는 제거
- 확장 복습 큐
- 상세 성장 리포트
- 즐겨찾기 또는 나만의 복습 목록
- 광고 없는 사용 경험
- 향후 추가되는 프리미엄 콘텐츠 우선 제공

MVP에서 프리미엄 기능은 개발 부담이 큰 기능보다 **콘텐츠 접근 범위와 복습 편의성** 중심으로 설계한다.

### 10.5 광고 정책

MVP에서는 광고를 기본 수익 모델로 삼지 않는다.

이유:

- 학습 흐름을 끊는다.
- 초기 사용자 경험을 저하시킨다.
- 낮은 트래픽에서는 수익 기여가 작다.
- 광고 SDK는 개인정보, 성능, 심사 이슈를 늘릴 수 있다.

필요 시 무료 플랜에 제한적 광고를 넣는 것은 후속 검토 대상으로 둔다.

## 12. 운영 전략

### 12.1 1인 개발자 운영 원칙

dash2zero는 1인 개발자 운영을 전제로 한다.

따라서 다음 원칙을 따른다.

- 기능보다 유지보수 가능성을 우선한다.
- 복잡한 관리자 페이지보다 단순한 콘텐츠 관리 구조를 우선한다.
- 사용자 기능에 AI를 직접 넣기보다 콘텐츠 제작 보조에만 쓴다.
- 음성 녹음, 커뮤니티, 실시간 채팅처럼 운영 리스크가 큰 기능은 미룬다.
- 고객지원은 앱 내 문의와 이메일 중심으로 단순화한다.
- 분석 이벤트는 적게 시작하되 핵심 퍼널은 반드시 측정한다.

### 12.2 콘텐츠 운영 방식

초기 콘텐츠 운영은 다음 구조가 적합하다.

1. 스프레드시트 또는 CSV로 단어 원본 관리
2. AI로 예문, 태그, 오답 후보 초안 생성
3. 사람이 한국어와 영어 품질 검수
4. 앱에서 읽을 수 있는 JSON 또는 DB 형태로 변환
5. 앱 배포 또는 원격 데이터 업데이트로 반영

초기에는 콘텐츠 어드민을 크게 만들지 않는다. 콘텐츠 품질과 앱 학습 흐름 검증이 먼저다.

### 12.3 고객지원 원칙

MVP 고객지원 범위:

- 결제 문의
- 구독 취소 안내
- 앱 오류 신고
- 콘텐츠 오류 신고
- 계정 삭제 요청
- 개인정보 관련 요청

MVP에서 제공하지 않는 지원:

- 개인별 한국어 첨삭
- 발음 평가
- 학습 상담
- 실시간 채팅 상담

| D7 리텐션 | 7일 내 반복 사용 비율 | 습관 가능성 |
| 일일 복습 완료율 | 복습 큐를 처리한 비율 | SRS 부담 적정성 |
| 단어 정답률 | 퀴즈 정확도 | 난이도와 학습 효과 |

### 13.2 학습 품질 KPI

### 13.4 MVP 의사결정 기준

MVP 출시 후 다음 질문에 답할 수 있어야 한다.

- 사용자가 첫 학습을 끝까지 완료하는가
- 사용자가 다음 날 다시 오는가
- 300단어 콘텐츠가 충분히 매력적인가
- 무료 제한이 너무 강하거나 약하지 않은가
- 사용자가 저가 프리미엄에 비용을 지불할 의사가 있는가
- 음성 녹음 없이도 초기 학습 가치가 전달되는가
- 어떤 단어팩이 가장 많이 소비되는가

### 14.2 손익 가정 (D-018 봉인 후 갱신)

월 구독 가격 $4.99 기준 (사용자 결제 후 Apple/Google 30% 수수료 제외 시 net $3.49/mo per user). 따라서 초기에는 큰 수익보다 다음 순서를 목표로 한다.

1. 사용자가 반복해서 쓰는 학습 루프 검증
2. 무료 사용자의 유료 전환 가능성 검증 (PRD §8.2 paywall_view_to_purchase 4% target / 2% minimum)
3. 월 고정비를 감당할 수 있는 최소 유료 사용자 확보
4. 콘텐츠 팩 확장을 통한 장기 구독 이유 확보 (starter 60 → core 180 → topik)

**Break-even 추정**:
- 월 운영 burn $10~30 + Apple/Google 연 $124 평균 → 월 $20~40
- Premium $4.99/mo × Apple 70% net = $3.49/user/mo
- **유료 사용자 6~12명 = break-even** (사업계획서 v0.1 가정 22명 대비 절반)
- 베타 30명 → paywall conversion 4% → 1~2명 (target) — break-even 미달, GA 후 1k MAU 도달 시 break-even 가능

**연 구독 영향**:
- Premium Annual $49.99/yr × Apple 70% = $34.99/year net
- 월 환산 = $2.92/mo (월 구독 대비 16% 낮음 — Apple 1년 후 수수료 15% 할인 미적용 가정)
- 연 구독 비중 50% 가정 시 평균 net = ($3.49 + $2.92) / 2 = $3.21/user/mo
- **연 구독 50% 가정 break-even = 유료 사용자 7~13명**

### 14.3 가격 조정 원칙

가격은 다음 조건에서 조정할 수 있다.

- 유료 전환율이 낮지만 사용성 지표가 좋으면 가격 또는 무료 제한을 조정한다.
- 리텐션이 낮으면 가격이 아니라 제품 가치를 먼저 개선한다.
- 연간 플랜 비중이 낮으면 연간 가격 또는 혜택을 조정한다.
- 콘텐츠가 충분히 확장되기 전까지 가격 인상은 신중하게 한다.

## 19. 결론

dash2zero의 초기 사업 전략은 넓게 이기는 것이 아니라 좁게 완성하는 것이다.

초기 제품은 한국어 전체를 가르치지 않는다. 회화를 완성하지도 않고, AI 튜터를 제공하지도 않으며, 발음을 평가하지도 않는다. 대신 영어권 초보자가 매일 부담 없이 열 수 있는 **3분 한국어 단어 학습 루프**를 완성한다.

MVP의 성공 여부는 다음 질문 하나로 판단한다.

> 사용자가 dash2zero를 통해 한국어 단어를 더 쉽게 다시 보고, 듣고, 기억하고, 다음 날 다시 돌아오는가?

이 질문에 긍정적으로 답할 수 있다면, 이후 발화 훈련, AI 개인화, 콘텐츠 확장, 더 정교한 커리큘럼으로 확장할 수 있다.

따라서 현재 단계의 최우선 목표는 다음과 같다.

> 영어권 초보자를 위한 300단어 기반 모바일 한국어 학습 MVP를 작고 완성도 있게 출시한다.
